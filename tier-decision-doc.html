What to show, what to bury:
a tiering model for 133 agent events

Eight QI agents stream ~133 distinct websocket/SSE events. The agents are fixed — we can only change the frontend. This doc proposes a five-tier display model, then puts it through a six-lens adversarial stress test plus a reconciliation pass, landing on a defensible per-event verdict for engineering sign-off.

The model

Five tiers, one rule each

Every event lands in exactly one lane based on a single question: what is the user's job with this information? Lifecycle (start/progress/end/standalone) decides whether a tier opens a card, updates it, or closes it — it is the verb; the tier is the prominence.

Suppress
—
Plumbing & dead. Hidden, or behind a dev/verbose toggle. token_usage, context saves, inbound routing.
Murmur
—
One live, breathing status line — replaced in place, never the raw stream. reasoning, progress.
Milestone
—
The glanceable, collapsible step spine. Failures are a card state, not a new lane. plan_*, tool_*, sub_agent_*.
Outcome
—
Persistent, actionable cards + the accept/reject gates. test_case_saved, question_asked, charts.
Chat
—
Not an activity tier — conversation bubbles. message, user_message, dev_context.

Decision map

Where the 133 events land

Distribution after all rulings. The spine (Milestone) is the largest lane by design — but the stress test found it was originally over-stuffed, so tool-open and legacy phase ticks were pushed down to Murmur, and the legacy run-close was lifted up to Outcome.

The headline finding. The five-tier model is sound for the three typed agents (COMMON, TAA, TPA-v2). Its load-bearing risk is the legacy half (CAA, RAA, TPA-v1) where lifecycle/group_type are inferred, not in the wire — and the catch-all event envelope, which can smuggle a chart or a gap-finding into what looks like a throwaway status line. Both demand a normalization layer the data does not yet provide. See mandates.
Method

Six lenses tried to break it

Each lens independently scrutinised the proposed Tier column from a different angle, returned reclassifications with severity, then a reconciliation judge re-voted on every event where the lenses disagreed.

⚙️
Frontend feasibility
Implementable, but not as-is. Every Murmur/Milestone promise for the 3 legacy agents silently assumes a normalization layer. Biggest blocker: the event envelope as flat Murmur will drop user-facing charts.
🧠
Cognitive load
Spine over-pinned. Tool-call pairs + phase/plan overlap turn Milestone into a scrolling wall, while findings & non-terminal errors scroll away. Move tools to Murmur, promote findings/errors.
💥
Data-shape adversary
Brittle against the legacy 50%. Two transport shapes (type vs entry_type), divergent option/viz/sub-agent payloads, and out-of-order arrivals will mis-tier ~¼ of streams without defensive rules.
♿
Accessibility
Sound skeleton, two P0 gaps. Failures & pauses are described as silent visual restyles — strands AT users in undetectable deadlocks. Needs assertive-failure rule + gate focus management.
🔬
Power-user density
Serves novices, starves experts. Conflates "hide by default" with "discard" — destroying reasoning trace, validation issues, tracebacks & cost. Fixed by one Dev toggle + inspect-on-demand.
🧩
Cross-agent consistency
Highly consistent except the close. Sub-agent lifecycle, saves, gates & plumbing are uniform. The one P0: legacy session_complete is Milestone while TAA's close is Outcome — on one shared socket.
⚖️
Reconciliation judge — binding ruling
Resolved 9 contested events, mostly via hybrids that satisfy multiple lenses instead of binary picks: tier-by-payload (validation_completed, event), open-vs-settle timing (tool_started), and default-tier-plus-reveal (everything power-users wanted). A feasibility P0 (un-renderable) trumps a UX preference; a cognitive-load P0 (wall of noise) trumps a consistency nicety.

The router

One pass, top to bottom

The render layer resolves every frame through this order. Normalization runs first (it must), then tier routing, then the dev/inspect reveals layer on top of any tier.

0. NORMALIZE  wire = msg.type==='stream_entry' ? msg.entry_type : msg.type
            flatten payload/metadata · backfill lifecycle & group_type for legacy
            coalesce label ??= sub_agent · options→[{id,label}] · viz: payload.data ?? payload

1. Is it transport / plumbing / inbound routing?
   └─ yes → SUPPRESS   (token_usage, context_saved, *_ticket, github_pr, replay_complete…)
            └─ but expose in DEV MODE if it carries cost/provenance (token_usage, sprint_issues_loaded)

2. Is it a conversation bubble? (message, user_message, dev_context)
   └─ yes → CHAT          (welcome = greeting bubble — dedupe per thread so reconnects don't re-greet)

3. Generic `event` envelope? → route on inner metadata.event_type
   ├─ visualization / *_detected / *_complete → OUTCOME
   └─ else                                     → MURMUR

4. Durable artifact, gate, or run-close briefing?
   └─ yes → OUTCOME   (*_saved, visualization, question(_asked), phase_review,
                       summary_generated, legacy session_complete, PR-TAA result + error)
            gates take focus & pause · validation_completed→Outcome only on fail

5. High-frequency tick / thought / live progress?
   └─ yes → MURMUR     (reasoning, progress, *_iteration, *_auto_progress,
                       legacy phase, plan_appended, tier_view_built, ladder_rung_committed,
                       tool open-phase — settles into Milestone on completion)

6. Otherwise — a discrete step in the run
   └─        MILESTONE  (session_*, plan_*, phase_*, tool close, sub_agent_* lifecycle,
                       skill_loaded, scoring_completed, tms_backfill_*)
            failures = card state + aria-live=assertive · traceback redacted (dev-only)
Non-negotiable

Top 5 implementation mandates

Ranked by severity. Without #1 and #2, the model mis-tiers roughly a quarter of real streams.

1
Normalization layer (P0, hard dependency). Before the UI sees anything: unwrap the stream_entry/entry_type envelope, backfill lifecycle + validate group_type for legacy literals, and normalize divergent shapes (sub_agent↔label, bare-string↔{id,label} options, flat↔nested viz). Nothing downstream renders correctly without it.
2
Event-router for the generic event envelope (P0). Dispatch each event by inner metadata.event_type: visualization / *_detected / *_complete → Outcome, everything else → Murmur. A flat Murmur tier demonstrably drops charts and gap findings.
3
Terminal-winner & card-close guarantee. Pick one visible close per agent (so agent_done + session_completed + summary don't triple-render). Every opened card must close on its lifecycle end keyed by group_id (not tool_call_id, which RAA omits) — re-home the latent validation_failed / tool_complete to Milestone so a stray completion still settles its card.
4
Gate focus & assertive announce. Outcome gates (question/question_asked, phase_review, validation-fail, PR-TAA terminal) take focus and announce the pause; failures fire aria-live="assertive"; tracebacks are always redacted from the visible UI.
5
Session-sticky Dev mode + per-card inspect (frontend-only). One toggle reveals the reasoning trace, validation_iteration.issues[], error.traceback, token_usage cost/latency, full result_preview/args_preview, and skill_description — a render-layer reveal, never an event-schema change.
Beyond tiering

Four problems the tier model leaves open

Tiering decides which lane an event enters. It does not decide what happens when many events fight for one lane, when two agents close in one turn, when an unrecognised event arrives, or when one lane floods. Those are design decisions — resolved here, two of them with playable widgets, because contention and density are exactly what a static doc can't show.

1Murmur has many feeders but one slotbiggest gap

A tool is open while reasoning and a sub_agent_progress are also firing. Who owns the line? Without arbitration it thrashes, or a thought overwrites the live action.

Decision — Murmur is one line with up to three fixed segments: [ACTION] · [DETAIL] · [THOUGHT], each owned by a different priority band. A live action (tool-open or a determinate sub-agent step) always holds ACTION and can never be overwritten by a thought; reasoning is demoted to THOUGHT and is the first thing dropped under width pressure. Each channel holds its segment for a short TTL after its last frame, then yields; when nothing is live, the line clears. Parallel fan-out replaces ACTION with an N-of-M worker strip; a stall pins a banner above the line instead of competing for it.

Backend ask: stable tool_call_id on RAA's tool_start + a per-issue key for exact stall detection (frontend works without, with TTL/string-match heuristics).

2Two agents, one socket, one turn — how many closes?design decision

SCA forwards the union of TAA + CAA into one thread. Both can emit an Outcome-tier run-close in a single turn. Two closes, or one merged sprint close?

Decision — Separate, source-attributed closes under one turn-level cap — not a backend merge. The two briefings answer different questions ("coverage done" vs "authoring done") and stay distinct, durable cards. The frontend renders an agent-grouped spine — one chronological timeline where contiguous same-source events collapse into source-labelled bands (◈ Coverage / ◇ Authoring), keyed off the source tag — then composes a thin, non-authoritative turn cap ("Sprint coverage turn complete · 14 cases · 3 scenarios · 4 analyses · 7 gaps"). The cap is suppressed when only one agent ran.

3The event router fails quietly on its open tailsilent data loss

*_detected/*_complete/visualization → Outcome, else Murmur. But event_type is runtime & unbounded — a future gap_found misses the suffix and silently falls to Murmur, dropping the very chart the rule exists to protect.

Decision — Route fail-VISIBLE, by payload shape, not by name suffix. A layered ladder: (0) transport guard → Suppress; (1) explicit allowlist → tier; (2) payload-shape sniffing — the real safety net: chart-shaped / findings-bearing / artifact-shaped → Outcome regardless of name; (3) suffix as a weak upgrade-only hint; (4) safe default — any payload-bearing unknown becomes a persistent "Unrecognised result" card, never ephemeral Murmur. Every allowlist miss fires a deduped telemetry beacon, and a backend namespacing ask makes routing eventually deterministic.

Routing ladder
  • L0 · transport / plumbing guard → Suppress
  • L1 · explicit allowlist (agent:event_type) → tier
  • L2 · payload-shape sniff (chart / findings / artifact) → Outcome
  • L3 · suffix hint (upgrade-only, never demotes)
  • L4 · safe default → "Unrecognised result" card / log line
Backend ask — REQUIRED: a top-level display_tier hint (outcome|milestone|murmur|suppress|chat) the emit site sets once — collapses the whole heuristic. NICE-TO-HAVE: a reserved event_type prefix convention (out.* / viz.* / sig.* / step.* / log.* / sys.*) — prefixes are unambiguous where suffixes aren't. Telemetry: event_router.unmatched_type beacon (type · agent · shape-fingerprint · resolved tier · routed-by), first occurrence sent immediately, rest aggregated every 60s — so a new type is discovered in hours, then promoted from shape-routed → allowlisted in one release.
4Outcome cards flood the wall14+ cards, 14+ decisions

A regression run emits 14+ test_case_saved cards — each correctly Outcome, but together a wall, and 14+ accept/reject decisions.

Decision — Summary-first, grouped, collapse-past-N, review-then-bulk-accept. Land on a pinned summary banner with a triage CTA; nest cases under their scenario; collapse groups past 3 and cards past 5; stream live into an accumulating group counter (no scroll-yank); turn 14 decisions into 1–3 clicks (Accept all / Accept P0-P1 / per-scenario) with per-card override; reconcile out-of-band acceptance_status_updated by test_case_id.

Thresholds: groups collapse past 3 (only the active/first expanded), cards collapse past 5 with a "+N more" chip, a hard ceiling at 50 forces header-only triage. Live: a rolling window of the last 3 authored cards keeps a streaming group ≤4 tall; no autoscroll if the user scrolled up (a "↓ N new" pill instead). Default = review-then-bulk-accept — nothing auto-accepts, nothing is pre-selected. Edges: lazy-parent placeholder for a test_case_saved that beats its scenario_saved; rejected cards demote out of the open-card budget; groups virtualise (collapsed = one header row, no child DOM).
